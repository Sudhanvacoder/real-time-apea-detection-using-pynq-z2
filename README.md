# real-time-apea-detection-using-pynq-z2
 Real-time Edge AI for neonatal apnea on PYNQ-Z2. Detects 20s breathing pauses via a 1D-CNN (Verilog RTL) and AXI4-Lite. Features a custom 3-tap systolic architecture (w  0 ​  =2,w  1 ​  =3,w  2 ​  =4) and C++ control in Vitis. Low-latency, cloud-free monitoring for NICUs. Validated on hardware via Tera Term. 
Real-Time Edge AI Hardware Accelerator for Neonatal Apnea DetectionThis project implements a low-latency, edge-based AI solution for detecting neonatal apnea—a critical condition where breathing ceases for more than 20 seconds. The system leverages a 1D-Convolutional Neural Network (1D-CNN) hardware accelerator implemented on an AMD/Xilinx PYNQ-Z2 FPGA.🏥 Background & MotivationWhat is Neonatal Apnea?Apnea of Prematurity (AOP) occurs when a newborn stops breathing for 20 seconds or longer. This is common in premature infants whose neurological systems are not yet fully developed.Why is Real-Time Detection Critical?Neurological Safety: Prolonged oxygen deprivation can lead to long-term developmental issues.Immediate Intervention: The standard treatment is physical stimulation (e.g., rubbing the infant's feet) to restart breathing. Every second counts.Edge AI Reliability: By using an FPGA at the "edge," we eliminate cloud latency and ensure 24/7 monitoring that doesn't crash if the internet fails. This provides a constant, reliable safety net for the Neonatal Intensive Care Unit (NICU).🚀 OverviewThe system processes respiratory signals in real-time using a custom Verilog-based RTL core to offload inference from the ARM processor, ensuring high-speed detection and immediate clinical alerts.Target Device: PYNQ-Z2 (Zynq-7000 SoC)Architecture: 3-tap 1D-CNN (Systolic Architecture)Design Flow: Vivado-Vitis Hardware-Software Co-design🛠 Project WorkflowI developed this system through a full-stack hardware-software integration process:1. Model Development & QuantizationDeveloped a 1D-CNN model using Python and Keras to identify apnea patterns in respiratory data.Extracted and quantized the weights (specifically $w_0=2$, $w_1=3$, $w_2=4$) to allow for efficient fixed-point arithmetic within the FPGA fabric.2. RTL Design (Hardware)Implemented the 3-tap 1D-CNN using a Systolic Architecture in Verilog.Designed the logic to handle the sliding window convolution across the incoming data stream.Wrapped the RTL in an AXI4-Lite interface to allow the ARM processor to communicate with the hardware core.3. SoC Integration (Vivado)Integrated the custom Verilog IP into the Zynq Processing System (PS) using Vivado IP Integrator.Configured the memory mapping and address space for the CNN core.Generated the bitstream and exported the hardware platform (.xsa).4. Embedded Software Development (Vitis)Developed the control application in C++ within the Vitis environment.Implemented the 20-second clinical timer logic: if the CNN score indicates a lack of breathing for 20 continuous seconds, an alert is triggered.Coded the data feeding mechanism to simulate real-time input to the FPGA core and read back the inference scores.5. Hardware ValidationDeployed the bitstream and software to the PYNQ-Z2 board.Validated the real-time detection performance using Tera Term to monitor the serial output and confirm the 20-second alert triggers correctly.📁 Repository Structure/rtl: Verilog source files and AXI wrappers./vitis: C++ source code for the ARM processor./vivado: Block designs and hardware export files./docs: Technical documentation and design diagrams.👤 ContributorSudhanva Raichur – Nitte Meenakshi Institute of Technology (NMIT)
